Mark heeft 2026-05-11 expliciet gevraagd: zet in CLAUDE.md dat je soms in memory moet kijken om taken af te werken. Reden: niet álle openstaande items komen in `docs/AUTONOOM-BACKLOG.md` terecht — sommige strategische beslissingen + uitgestelde projecten worden in memory opgeslagen omdat ze sessie-overstijgend zijn.

**Wanneer memory raadplegen**:
- Bij sessie-start (na CLAUDE.md lezen).
- Bij "ga verder" als de backlog leeg of onduidelijk is.
- Bij beslissingen over richting/scope (memory bevat eerdere keuzes).
- Bij twijfel of een feature al besloten/uitgesteld is.

**Wat te zoeken**:
- Bestanden met prefix `project_` — openstaande projecten/roadmaps.
- Bestanden met prefix `feedback_` — beslissings-rules.
- Specifiek: zoek op "TODO", "later", "uitgesteld", "C-taak", "open" in memory-files.

**Why**: Mark wil dat tijdverbruik laag blijft. Hij wil niet 3 keer hetzelfde besluit nemen. Memory is GEEN dood archief — het is een actieve werklijst voor groot/strategisch werk.

**How to apply**:
- Behandel memory als TWEEDE backlog naast `docs/AUTONOOM-BACKLOG.md`.
- Bij elke nieuwe sessie even MEMORY.md scrollen op `project_*` en zoeken naar nog-niet-afgevinkte items.
- Als Mark om "een goed pad voor vandaag" vraagt — eerst backlog, dan memory checken.
